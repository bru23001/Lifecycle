# Appendix B — Checklists

This appendix is the supplementary **UI/UX checklist library** for the Master Lifecycle. It does not replace lifecycle-wide quality/compliance checks in **`25. Quality and Compliance Checks.md`** or normative USSM Annex E checklists.

Normative compliance review checklists for SRS, design, and testing remain in **USSM Section 10.5 — Annex E**. This appendix holds UI/UX scoring and audit criteria for design reviews (Phase 7) and experiential QA (Phase 10).

**Layout patterns (CRM / admin):** For concrete shell structure (global containers, 12-column grid, sidebar + content, master–detail, modal overlay, section wireframes for lists/dashboard/RBAC/CMS), use **`UI-UX Layout Guide — LYG-001.md`** in addition to the scoring tables below.

## UI/UX Scoring Criteria and Audit Framework

**Classification:** Keep — standalone Appendix B UI/UX scoring and audit framework for visual hierarchy, consistency, feedback, typography, usability heuristics, information architecture, accessibility, and task-friction metrics.

When evaluating **UI** (visual and interactive surface), criteria emphasize **how it looks and responds**, balancing aesthetics with functional clarity.

### UI — Visual hierarchy and composition

| Topic | What to score |
| --- | --- |
| **Contrast** | Text and controls meet readability targets (e.g. WCAG 2.1 — commonly **4.5:1** for normal body text unless a conformant exception applies). |
| **Whitespace** | Enough negative space to avoid cognitive overload vs. cluttered layouts. |
| **Fitts’s law** | Primary actions are easy to acquire—adequate target size and placement. |

### UI — Consistency and standardized patterns

| Topic | What to score |
| --- | --- |
| **Component reuse** | Buttons, inputs, icons follow one system across screens. |
| **Internal consistency** | Repeated patterns (e.g. cancel/destructive placement) stay predictable. |
| **External consistency** | Alignment with platform conventions (e.g. Material Design, Apple Human Interface Guidelines) where applicable. |

### UI — Interactive feedback (“feel”)

| Topic | What to score |
| --- | --- |
| **Affordances** | Interactive elements read as interactive (buttons, links, tappable areas). |
| **States** | Clear cues for hover, active, disabled, loading. |
| **Perceived responsiveness** | Immediate acknowledgment (~**100 ms** perceived feedback) even when backend work is slower. |

### UI — Typography and readability

| Topic | What to score |
| --- | --- |
| **Hierarchy** | Distinct roles for headings vs. body vs. secondary text. |
| **Measure and rhythm** | Comfortable line length (often ~**45–75** characters per line for body copy) and spacing. |

### Quick UI audit scorecard (example dimensions)

| UI attribute | Goal | Example metric |
| --- | --- | --- |
| Aesthetics | Professionalism and brand fit | Subjective rating (e.g. 1–10) |
| Accessibility | Inclusivity | Contrast / typography pass–fail vs. policy |
| Efficiency | Speed of interaction | Click/tap accuracy in moderated tasks |
| Clarity | Reduced confusion | Icon or label recognition |
| Feedback | System status | Loading/error/success patterns present |

## UX — Journey, friction, and structure

**UX** separates **logic, friction, and emotion** of the journey from pure visuals.

### Core usability (heuristic lens)

Score severity of issues (e.g. **0–4**) across typical heuristic themes:

- **Recognition vs. recall** — Is needed information visible vs. memorized across steps?
- **Flexibility and efficiency** — Shortcuts for experts without harming novices?
- **Error prevention** — Guardrails before destructive or irreversible actions?
- **User control and freedom** — Undo, escape paths, recoverability?

### Information architecture (IA)

| Measure | Intent |
| --- | --- |
| **Tree testing** | Users can find targets from navigation labels alone (text-only IA probe). |
| **Card sorting agreement** | Categories match user mental models. |
| **Search vs. navigation** | Heavy immediate search use may signal navigation or labeling gaps. |

### Experience qualities (Morville honeycomb)

Useful · Usable · Desirable · Findable · Accessible · Credible · Valuable — qualitative scoring across facets as appropriate to the product.

### Interaction / friction metrics

| Metric | Signal |
| --- | --- |
| **Task satisfaction / mental effort** | E.g. single-task effort rating after representative tasks. |
| **Back navigation usage** | Elevated “back” use may indicate confusion or wrong paths. |
| **Pogo-sticking** | Search/catalog: click result → immediate return → another result suggests relevance or preview gaps. |

### UI vs. UX emphasis (same surface, different lens)

| Element | UI emphasis | UX emphasis |
| --- | --- | --- |
| Button | Visual treatment, affordance | User understands outcome before acting |
| Form | Legibility, density | Field count, flow, errors recoverable |
| Navigation | Visual cleanliness | Primary destinations discoverable |
| Error | Styling, visibility | Message explains fix or next step |

**Specialized systems** (e.g. dense dashboards): often weight **efficiency** and **error prevention** highest.
