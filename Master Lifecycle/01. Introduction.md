# Introduction

This procedure family defines how CYBERCUBE selects, plans, builds, tests, releases, and maintains software across the full software development lifecycle.

The procedure family is intentionally modular. Numbered phase documents define operational lifecycle activities, required outputs, completion criteria, and approval expectations. Overlay manuals provide additional normative guidance where deeper standards, controls, or implementation rules are required.

The **Unified Software Standards Manual (USSM) v1.0** (`USSM — Unified Software Standards Manual v1.0.md`) shall serve as the primary SDLC documentation backbone for producing, reviewing, and auditing CYBERCUBE software project artifacts. USSM aligns the lifecycle with **ISO/IEC 12207** and **IEEE 29148** and defines the expected documentation hierarchy:

CRS → SRS → DDS/SDD → development artifacts → test artifacts → deployment records → maintenance records.

Use **`03. Scope.md`** to determine applicability, lifecycle boundaries, included project types, and excluded activities.

Use **`04. Definitions.md`** for controlled terms (gates, traceability, overlays, tailoring, USSM documentation tiers).

Use **`06. Lifecycle Overview.md`** to understand how lifecycle phases, approval gates, required outputs, and USSM sections map together.

Use **`21. Decision Gates.md`** for the gate model and evidence expectations at phase transitions.
